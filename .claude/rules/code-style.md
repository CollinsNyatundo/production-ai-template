# Code Style Rules

Coding assistants and human developers must follow these formatting and design standards:

* **Format:** Use Black for code styling and formatting. Auto-apply prior to code commit.
* **Imports:** Sort imports using `ruff` or `isort`. Group by standard library, third-party libraries, then local project modules.
* **Variable Naming:** Use snake_case for functions and variables. Use PascalCase for classes. Use UPPERCASE for global configurations.
* **Docstrings:** Standardize docstrings utilizing Google Python Style Guide format. Always provide docstrings explaining parameter definitions and return shapes.
